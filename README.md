## 💡 El Problema y la Filosofía del Proyecto
La brecha digital es uno de los mayores desafíos en la modernización de negocios familiares o tradicionales. Este proyecto fue codificado a solicitud de una persona mayor (dueño de la distribuidora) con el objetivo firme de dejar atrás los cuadernos, el papel y el conteo manual. 
El verdadero desafío en el desarrollo Backend y Frontend aquí **no fue construir el sistema más complejo, sino el más intuitivo**. Cada vista y cada proceso de base de datos fue pensado para que la curva de aprendizaje del cliente fuera mínima, garantizando una transición al mundo digital sin frustraciones.
## 🛠️ Stack Tecnológico
El proyecto se estructuró buscando ser ligero, de bajo mantenimiento y autogestionable a largo plazo:
- **Backend & Enrutamiento:** `Python` + `Flask`. Un framework robusto y sin exceso de equipaje que expone de forma directa las rutas del sistema logístico.
- **Base de Datos:** `SQLite`. Implementación de bases de datos embebidas. Ligereza de lectura masiva sin necesidad de mantener servicios de bases de datos externos.
- **Frontend & UI:** HTML modular potenciado con **Flowbite (Tailwind CSS)**. Se usó este sistema de diseño porque otorga componentes grandes, legibles, contrastados y visualmente familiares. Ideal para usuarios sin perfil tecnológico.
## ⚙️ Características Clave
1. **Gestión de Carga Limpia:** Formularios y tablas sin sobreinformación, yendo directamente a los datos vitales que el negocio necesita.
2. **Optimización de Interacciones:** Acciones administrativas a un máximo de 2 'clicks' de distancia basándose en UX/UI amigable para usuarios adultos mayores.
3. **Persistencia Autónoma:** La instancia procesa todas las transacciones localmente bajo una estructura de SQLite segura y transaccional, evitando pérdida de datos contables por errores humanos.
